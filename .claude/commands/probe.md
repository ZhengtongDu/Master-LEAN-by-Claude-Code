# Probe - 解答学习问题并完善教程

检查 MyQuestions.md 中未勾选的问题，逐一解答，并在用户理解后询问是否将内容补充进教程。

## 执行流程

1. **读取问题列表**
   - 读取 MyQuestions.md 文件
   - 识别所有未勾选的问题（`- [ ]` 格式）
   - 按章节顺序列出所有待解答的问题

2. **逐一解答问题**
   - 从第一个未勾选的问题开始
   - 提供详细、清晰的解答
   - 使用示例代码（LEAN 代码）辅助说明
   - 确保解答符合教程的定位（研究生水平，有高等数学和编程基础）

3. **确认理解**
   - 解答完一个问题后，询问用户是否理解
   - 如果用户还有疑问，继续深入解答
   - 如果用户理解了，进入下一步

4. **询问是否补充进教程**
   - 询问用户是否希望将这个解答补充进对应章节
   - 如果用户同意：
     - 将解答内容整合进对应的章节文件（chapter*.md）
     - 在 MyQuestions.md 中勾选该问题（`- [x]`）
     - 将问题移动到"已解决的问题归档"区域
   - 如果用户不同意：
     - 仅在 MyQuestions.md 中勾选该问题
     - 不修改教程内容

5. **继续下一个问题**
   - 重复步骤 2-4，直到所有问题都被解答
   - 如果没有未勾选的问题，提示用户所有问题已解决

6. **总结**
   - 列出本次解答的问题数量
   - 列出补充进教程的内容数量
   - 提示用户可以继续添加新问题

## 注意事项

- 解答要深入浅出，符合教程的语言风格（中文）
- 提供的代码示例要可以在 LEAN 4 中运行
- 补充进教程时，要注意保持章节的连贯性和结构
- 不要一次性解答所有问题，而是逐个确认用户理解后再继续
- 如果问题涉及教程尚未覆盖的内容，可以建议用户先学习相关章节

## 示例交互

```
Claude: 我在 MyQuestions.md 中发现 3 个未解答的问题：

3. [第五章] 如何理解 Curry-Howard 同构？

让我们从第一个问题开始...

[详细解答]

你理解了吗？是否还有疑问？

用户: 理解了

Claude: 很好！是否希望我将这个解答补充进第一章的内容中？这样可以让教程更加完善。

用户: 是的

Claude: 好的，我会将解答整合进 chapter01.md，并更新 MyQuestions.md。

[执行操作]

现在让我们继续第二个问题...
```

开始执行 probe 命令。
