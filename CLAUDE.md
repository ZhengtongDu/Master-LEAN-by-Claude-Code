## 写作规范

- **语言**: 所有文档使用中文撰写，交流可以用英文
- **格式**: 使用 Markdown 格式
- **内容定位**: 要经常写一些toy case的代码，专注于讲解 LEAN 的原理、使用技巧
- **文本风格**：对于代码需要有详细的注释，讲解每个内容的动机，适合初学者阅读
- **目标**: 帮助读者从入门到精通 LEAN 定理证明器

### 格式润色准则（章节整理时首要参考）

1. **本章目标**：每章开头使用 `> **本章目标**：` blockquote + 编号列表格式，列出 3-5 个学习目标，每项用加粗关键词开头。
2. **概念 blockquote**：每个主要小节（##/###）的核心概念使用 `> **概念名**：` 格式的 blockquote 引出，替代纯文本段落式的概念定义。
3. **契约注释**：所有 `def` / `theorem` 定义前添加 `/-- 名称: 描述 输入: ... 输出: ... -/` 格式的文档注释。
4. **代码块标记**：所有代码块必须使用 ` ```lean ` 语言标记。
5. **段落间距**：`##` / `###` 标题前后必须有空行；blockquote 与正文之间必须有空行；**Toy Case** 标签前后必须有空行。
6. **小结格式**：章末使用 `## 本章小结` 标题 + 编号列表 + 加粗关键词格式。
7. **分隔线**：主要章节（##）之间使用 `---` 分隔。
8. **代码正确性**：确保所有代码使用标准 Lean 4 策略/API，不使用非标准或已废弃的策略名（如 `false_or_by_contra`），优先使用 `exact h.elim` / `exact False.elim h` / `cases h` 等标准写法。
9. **避免重复**：blockquote 中已涵盖的概念解释不要在正文中重复。
